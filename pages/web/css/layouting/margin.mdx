# Margin

`margin` property sets the margin area of an element (the distance between the element and adjacent elements).

<Image path="/res/web/css/box-model.png" caption="Illustration of CSS Box model" />

## Common Margin Properties

### `margin`

Specifying `margin` allows you to set the top, right, bottom, and left margins of an element (remember: clockwise direction).

If you specify less than 4 margin lengths, the missing lengths would be inferred from the lengths that you specified (it will try to make symmetrical margins).

Possible length values:

### `margin-top`

Sets the element's top margin. (See `margin`.)

### `margin-right`

Sets the element's right margin. (See `margin`.)

### `margin-bottom`

Sets the element's bottom margin. (See `margin`.)

### `margin-left`

Sets the element's left margin. (See `margin`.)

## Obscured Margin Properties

### `margin-inline-start`

Sets the margin at the start of "inline" direction. If you're writing horizontally from left to right, then it maps to `margin-left`.

(Where this direction is depends on `writing-mode`, `direction`, and `text-orientation`. See `margin` for the list of possible values.)

### `margin-inline-end`

Sets the margin at the end of "inline" direction. If you're writing horizontally from left to right, then it maps to `margin-right`.

(The "inline" direction depends on `writing-mode`, `direction`, and `text-orientation`. See `margin` for the list of possible values.)

### `margin-inline`

Shorthand for `margin-inline-start` and `margin-inline-end`.

```css
p {
  margin-inline: <margin-inline-start> <margin-inline-end>;
  margin-inline: <margin-inline-start>;
}
```

### `margin-block-start`

Sets the margin at the start of "block" direction (perpendicular to "inline" direction). If you're writing horizontally from left to right, top to bottom, then it maps to `margin-top`.

(Where this direction is depends on `writing-mode`, `direction`, and `text-orientation`. See `margin` for the list of possible values.)

### `margin-block-end`

Sets the margin at the end of "block" direction (perpendicular to "inline" direction). If you're writing horizontally from left to right, top to bottom, then it maps to `margin-bottom`.

(Where this direction is depends on `writing-mode`, `direction`, and `text-orientation`. See `margin` for the list of possible values.)

### `margin-block`

Shorthand for `margin-block-start` and `margin-block-end`.

```css
div {
  margin-block: <margin-block-start> <margin-block-end>;
  margin-block: <margin-block-start>;
}
```

## Margin Collapsing

When _vertical_ margins (top and bottom) of different elements touches each other, the margin "collapses" into one single margin.

**It does not apply to:**

- horizontal margins (left and right),
- margins of floating elements, and
- margins of absolutely positioned elements.
