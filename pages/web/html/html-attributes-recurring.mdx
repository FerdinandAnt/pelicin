# Recurring HTML Attributes

Some HTML attributes are recurring in more than one type of elements. They are described here.

## `crossorigin`

Defines how the browser should make the request in terms of CORS. Possible values:

<ul>
  <li>
    <strong>
      <code>anonymous</code>
    </strong>{' '}
    &mdash; <em>Do not</em> send credentials for CORS when requesting the resource. Expect the
    server to allow CORS via <code>Access-Control-Allow-Origin</code> response header
  </li>
</ul>

<ul>
  <li>
    <strong>
      <code>use-credentials</code>
    </strong>{' '}
    &mdash; Send credentials for CORS when requesting the resource. Expect the server to allow CORS
    via <code>Access-Control-Allow-Credentials</code> response header
  </li>
</ul>

<strong>Note:</strong> <dfn>Credentials</dfn> are cookies, authorization headers, or TLS client certificates.

## `referrerpolicy`

Whether or not the browser should include the `Referer` header when making the request. Possible values:

<Badge color="red">No-referrer</Badge>
<ul>
  <li>
    <strong>
      <code>no-referrer</code>
    </strong>{' '}
    &mdash; Don't send referer
  </li>
</ul>
<ul>
  <li>
    <strong>
      <code>no-referrer-when-downgrade</code>
    </strong>{' '}
    &mdash; (This is default.) Don't send referer if we are making request to less secure protocol
    (i.e. HTTPS to HTTP). Otherwise, send origin, path, and querystring as <code>Referer</code>.
  </li>
</ul>

<Badge color="red">Origin</Badge>
<ul>
  <li>
    <strong>
      <code>origin</code>
    </strong>{' '}
    &mdash; Only send origin as <code>Referer</code>
  </li>
</ul>
<ul>
  <li>
    <strong>
      <code>same-origin</code>
    </strong>{' '}
    &mdash; Only send origin as <code>Referer</code> when making same-origin requests. Otherwise,
    don't send referer.
  </li>
</ul>
<ul>
  <li>
    <strong>
      <code>origin-when-cross-origin</code>
    </strong>{' '}
    &mdash; Only send origin as <code>Referer</code> when making cross-origin request. Otherwise,
    send origin, path, and querystring as <code>Referer</code>.
  </li>
</ul>

<Badge color="red">Strict origin</Badge>
<ul>
  <li>
    <strong>
      <code>strict-origin</code>
    </strong>{' '}
    &mdash; Only send origin as <code>Referer</code> when the protocol security level is at least
    the same. Otherwise, don't send referer.
  </li>
</ul>
<ul>
  <li>
    <strong>
      <code>strict-origin-when-cross-origin</code>
    </strong>{' '}
    &mdash; (1) Only send origin as <code>Referer</code> when making cross-origin request and the
    protocol security level is at least the same. (2) Send origin, path, and querystring as{' '}
    <code>Referer</code> when making same-origin request and the protocol security level is at least
    the same. (3) Otherwise, don't send referer.
  </li>
</ul>

<Badge color="red">Badass mode</Badge>
<ul>
  <li>
    <strong>
      <code>unsafe-url</code>
    </strong>{' '}
    &mdash; Send origin, path, and querystring as <code>Referer</code>.{' '}
    <em>(May leak private data over insecure HTTP.)</em>
  </li>
</ul>

## `rel`

Defines relation of the linked resource to the document.

The possible values depend on how `rel` was used:

- `<link rel="...">`
- `<a rel="...">` or `<area rel="...">`,
- `<form rel="...">`

### `rel` of `<link>`

<Badge color="red">Metadata</Badge>
<ul>
  <li>
    <strong>
      <code>author</code>
    </strong>{' '}
    &mdash; link to the author
  </li>
  <li>
    <strong>
      <code>canonical</code>
    </strong>{' '}
    &mdash; link to the preferred URL for this document
  </li>
  <li>
    <strong>
      <code>alternate</code>
    </strong>{' '}
    &mdash; link to the alternate representation of this document, e.g.
    <ul>
      <li>
        <code>{`<link rel="alternate" href="www.example.com/jp-jp" hreflang="jp-jp"`}</code>
      </li>
      <li>
        <code>{`<link rel="alternate" type="application/atom+xml" href="posts.xml" title="Blog">`}</code>
      </li>
    </ul>
  </li>
  <li>
    <strong>
      <code>help</code>
    </strong>{' '}
    &mdash; link to help page of this document
  </li>
  <li>
    <strong>
      <code>license</code>
    </strong>{' '}
    &mdash; link to license page of this document
  </li>
  <li>
    <strong>
      <code>search</code>
    </strong>{' '}
    &mdash; link to search page of this document
  </li>
  <li>
    <strong>
      <code>next</code>
    </strong>{' '}
    &mdash; link to the next document in series
  </li>
  <li>
    <strong>
      <code>prev</code>
    </strong>{' '}
    &mdash; link to the previous document in series
  </li>
  <li>
    <strong>
      <code>pingback</code>
    </strong>{' '}
    &mdash; link to the{' '}
    <a href="https://wordpress.stackexchange.com/questions/116079/what-is-rel-pingback-and-what-is-the-use-of-this-in-my-website">
      pingback server
    </a>{' '}
    of this document (e.g., on Wordpress, if somebody writes a link to your post, Wordpress will try
    to contact the pingback server of your post to let you know that somebody has put down a link to
    your post)
  </li>
  <li>
    <strong>
      <code>manifest</code>
    </strong>{' '}
    &mdash; link to <a href="https://developer.mozilla.org/en-US/docs/Web/Manifest">PWA manifest</a>
  </li>
</ul>

<Badge color="red">Performance</Badge>
<ul>
  <li>
    <strong>
      <code>dns-prefetch</code>
    </strong>{' '}
    &mdash; tells the browser to preemptively perform DNS resolution for the target origin
  </li>
  <li>
    <strong>
      <code>preconnect</code>
    </strong>{' '}
    &mdash; tells the browser to preemptively connect to the target origin (this does DNS
    resolution, TLS negotiation, and TCP handshake)
  </li>
  <li>
    <strong>
      <code>prefetch</code>
    </strong>{' '}
    &mdash; tells the browser that it <em>should</em> fetch and cache the target resource because
    it's likely to be used in the followup navigation
  </li>
  <li>
    <strong>
      <code>preload</code>
    </strong>{' '}
    &mdash; tells the browser that it <em>must</em> fetch and cache the target resource because it's
    likely to be used in the current navigation
  </li>
  <li>
    <strong>
      <code>prerender</code>
    </strong>{' '}
    &mdash; tells the browser that it should fetch and process the target resource (e.g. fetch its
    subresources or perform rendering off-screen) because it's likely to be used in the followup
    navigation
  </li>
  <li>
    <strong>
      <code>modulepreload</code>
    </strong>{' '}
    &mdash; for JS modules: tells the browser to preemptively fetch the script (and its
    dependencies) and store it in the document's module map for later evaluation.
  </li>
</ul>

<Badge color="red">Resource type</Badge>
<ul>
  <li>
    <strong>
      <code>icon</code>
    </strong>{' '}
    &mdash; loads favicon
  </li>
  <li>
    <strong>
      <code>stylesheet</code>
    </strong>{' '}
    &mdash; loads CSS
  </li>
</ul>

## `target`

Targets where to open a link. Possible values:

<ul>
  <li>
    <strong>
      <code>_self</code>
    </strong>{' '}
    &mdash; current window/tab
  </li>
</ul>

<ul>
  <li>
    <strong>
      <code>_blank</code>
    </strong>{' '}
    &mdash; new window/tab
  </li>
</ul>

<ul>
  <li>
    <strong>
      <code>_parent</code>
    </strong>{' '}
    &mdash; Parent frame (otherwise behaves like <code>_self</code>)
  </li>
</ul>

<ul>
  <li>
    <strong>
      <code>_top</code>
    </strong>{' '}
    &mdash; Root frame (otherwise behaves like <code>_self</code>)
  </li>
</ul>
